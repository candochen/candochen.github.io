Scala的Currying 和 Partially Applied Function

Aug 21st, 2014 5:47 pm

今天看了王垠的一篇关于Currying的文章 谈谈Currying, 看完之后，我想知道Scala的Currying是否跟Haskel的相同，于是我查了查官网和其他文章，并做记录和分析如下。

Scala官网对Currying给出的定义是这样的：

Methods may define multiple parameter lists. When a method is called with a fewer number of parameters lists, then this will yield a function taking the missing parameter list as its arguments.

它说，一个函数可能有多个参数列表，对于一个拥有多个参数列表的函数来说，假如我们只传入部分参数列表去调用这个函数，那么这时候函数就会返回另外一个函数，这个函数的参数列表就是剩余的参数列表。举个例子，

def filter(xs: List[Int], p: Int => Boolean): List  [Int] =
    if (xs.isEmpty) xs
    else if(p(xs.head)) xs.head :: filter(xs.tail, p)
    else filter(xs.tail, p)
